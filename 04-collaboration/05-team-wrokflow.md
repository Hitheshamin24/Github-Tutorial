# Team Workflow in GitHub

## Basic Workflow

1. Create Issue
2. Create Branch
3. Work on feature
4. Push code
5. Create Pull Request
6. Code Review
7. Merge

---

## Example Flow

```
Issue → Branch → Code → PR → Review → Merge
```

---

## Best Practices

* Use meaningful commit messages
* Work in branches
* Do not push directly to main
* Review before merge

---

## Summary

Collaboration = Structured teamwork using GitHub
