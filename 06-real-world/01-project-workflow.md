# Real Project Workflow Using Git and GitHub

## Overview

This is how real teams work using Git and GitHub.

---

## Step-by-Step Workflow

1. Create Issue
2. Create Branch from main
3. Develop feature
4. Commit changes
5. Push to GitHub
6. Create Pull Request
7. Code Review
8. Merge to main

---

## Example Flow

```
Issue → Branch → Code → Commit → Push → PR → Review → Merge
```

---

## Key Insight

You NEVER work directly on main branch in real projects.

---

## Summary

Real workflow = Structured development + review process
